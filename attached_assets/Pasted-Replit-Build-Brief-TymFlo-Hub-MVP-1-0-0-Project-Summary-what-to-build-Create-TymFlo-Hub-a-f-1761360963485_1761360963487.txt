Replit Build Brief — TymFlo Hub (MVP 1.0)
0) Project Summary (what to build)

Create TymFlo Hub: a fast, SEO-driven web app that offers:

Business Calculators (profit margin, markup, break-even, ROI).

Document Tools (PDF merge/split/compress, image convert).

Productivity Utilities (timezone converter, Pomodoro timer).

Ship as a freemium product (5 free operations/day). Pro removes limits (Square payments). Must be PWA (offline shell), sub-2s load time, accessible, and SEO-optimized with one landing page per tool.

1) Tech Stack (opinionated to reduce choices)

Frontend: Vite + React + TypeScript (fast, simple), TailwindCSS (utility styling).

Backend: Node + Express (single server in Replit), TypeScript.

Data: SQLite via Prisma (simple, file-based), fallback to Replit DB if needed.

Auth: Passwordless magic link (email) via Magic.link OR simple email+OTP (local) for MVP.

Payments: Square (create checkout links; no vaulting in MVP).

File ops: pdf-lib for PDFs, sharp for images.

Rate limiting: express-rate-limit + IP/session.

Logging: pino.

Testing: Vitest (unit) + Playwright (smoke/E2E).

Analytics: Plausible or PostHog snippet.

PWA: service worker via vite-plugin-pwa.

5) Database Schema (Prisma /server/db/schema.prisma)
datasource db { provider = "sqlite"; url = "file:./dev.db" }
generator client { provider = "prisma-client-js" }

model User {
  id        String  @id @default(cuid())
  email     String  @unique
  createdAt DateTime @default(now())
  plan      Plan     @default(FREE)
  usage     Usage?
}

model Usage {
  id        String  @id @default(cuid())
  userId    String  @unique
  periodKey String   // e.g. "2025-10-24"
  ops       Int      @default(0)
  User      User     @relation(fields: [userId], references: [id])
}

enum Plan { FREE PRO TEAM }

6) Routing (server /server/router.ts)

POST /api/auth/magic/login → start magic link.

POST /api/auth/magic/callback → verify, set httpOnly cookie.

GET /api/usage → get today’s usage & limits.

POST /api/usage/increment → increment per tool op.

POST /api/tools/calculator → run calculators (type param: profitMargin, markup, roi, breakeven).

POST /api/tools/pdf/merge → accept up to 5 files, return merged file (stream).

POST /api/tools/pdf/split

POST /api/tools/pdf/compress

POST /api/tools/image/convert

GET /api/payments/checkout?plan=PRO|TEAM → returns Square checkout link.

POST /api/payments/webhook → upgrade plan on success.

GET /sitemap.xml (generated)

GET /robots.txt

Middleware: global CORS, JSON, rateLimit on /api/tools/*, auth required for Pro operations.

7) Usage Limits (freemium)

FREE: 5 operations/day (any tool).

PRO: unlimited + batch ops for PDFs/images.

TEAM: same as PRO + shared workspace (MVP: shared label only).
Implement usage.service.ts to:

Compute periodKey = YYYY-MM-DD.

Block with 429 if limit exceeded.

Client shows UpgradeNudge when limit hit.

11) Example: Calculator Logic (server/services/calculators.service.ts)
export function calcProfitMargin({ cost, price }: { cost:number; price:number }) {
  if (cost <= 0 || price <= 0) throw new Error("Invalid inputs");
  const profit = price - cost;
  const margin = (profit / price) * 100;
  return { profit, margin: Number(margin.toFixed(2)) };
}
export function calcMarkup({ cost, price }:{ cost:number; price:number }) {
  if (cost <= 0 || price <= 0) throw new Error("Invalid inputs");
  const profit = price - cost;
  const markup = (profit / cost) * 100;
  return { profit, markup: Number(markup.toFixed(2)) };
}


Client form (profit-margin.tsx) posts to /api/tools/calculator with { type:"profitMargin", cost, price }, shows result, increments usage.

12) Example: PDF Merge (server/services/pdf.service.ts)
